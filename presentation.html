1. What is this Project?

Supply Chain Tracker is a blockchain-powered web application that lets anyone register items, transfer ownership, and view full provenance history — all stored permanently and publicly on the Ethereum blockchain.

💡 Think of it like a permanent, unfakeable digital certificate for any product — nobody can edit or delete the history. Ever.

2. Real World Problem it Solves

The Problem with Traditional Supply Chains

  • Records are stored in private company databases — easily manipulated
  • Data can be edited or deleted at any time
  • No easy way to verify if a product is genuine
  • Middlemen can lie about product history

Example Scenario 🌿

A coffee farmer in Ethiopia sells coffee → exporter → distributor → supermarket → customer.

Who owned it at each step? Was it really organic? In traditional systems — you just have to trust. You can't verify.

Our Solution ✅

StepWhat Happens on Blockchain
Farmer registers batchItem added — farmer's wallet becomes permanent owner
Sells to exportertransferItem() called — new owner recorded forever
Exporter → distributorAnother transfer — history grows
Customer checksQueries by ID — sees every owner, in order, forever
Nobody can alter this history. Not even us. Not even the government. It's on the blockchain forever.

4. Smart Contract — The Heart of the App

The smart contract is the backend of this app. It runs on the blockchain — no company, no server, nobody controls it.

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SupplyChain {

    // Data Structure — what each item stores
    struct Item {
        uint id;            // unique number (1, 2, 3...)
        string name;        // product name
        address owner;      // current owner's wallet address
        address[] history;  // list of ALL past owners
    }

    mapping(uint => Item) public items;  // id → Item lookup
    uint public itemCount;               // total items counter

    event ItemAdded(uint id, string name, address owner);
    event ItemTransferred(uint id, address from, address to);

    // Function 1: Add a new item
    function addItem(string memory _name) public {
        itemCount++;
        items[itemCount] = Item(itemCount, _name, msg.sender, new address[](0));
        items[itemCount].history.push(msg.sender); // creator = first owner
        emit ItemAdded(itemCount, _name, msg.sender);
    }

    // Function 2: Transfer ownership (only owner can call this)
    function transferItem(uint _id, address _to) public {
        require(items[_id].owner == msg.sender, "Not owner"); // security!
        items[_id].owner = _to;
        items[_id].history.push(_to); // record new owner in history
        emit ItemTransferred(_id, msg.sender, _to);
    }

    // Function 3: Read full ownership history (free, no gas)
    function getHistory(uint _id) public view returns (address[] memory) {
        return items[_id].history;
    }
}

Code Explanation

CodeSimple Meaning
struct ItemA blueprint — what info each item stores
mapping(uint => Item)A dictionary: number → item data
msg.senderThe wallet address of whoever calls the function
require(...)Security guard — rejects transaction if condition fails
history.push(_to)Permanently adds new owner to the trail
public viewRead-only function — completely free, no gas
emit ItemAdded(...)Fires a blockchain event — apps can listen for it

Part 1 — Connecting to Ethereum (Provider)

import { BrowserProvider, Contract } from 'ethers'

// Creates a connection to MetaMask (the user's wallet in browser)
const provider = useMemo(() => {
  if (typeof window !== 'undefined' && 'ethereum' in window) {
    return new BrowserProvider((window as any).ethereum)
    //         ↑ reads from MetaMask, connects to blockchain
  }
  return null
}, [])
BrowserProvider is like a phone line from our React app to the Ethereum network, going through MetaMask.

Part 2 — Sending a Transaction (Add Item)

const addItem = async () => {
  // Step 1: Get the signer (user's wallet — ethers v6 is async!)
  const signer = await provider.getSigner()

  // Step 2: Connect to the smart contract with signer
  const contract = new Contract(contractAddress, abi, signer)

  // Step 3: Call the smart contract function → MetaMask popup appears
  const tx = await contract.addItem(itemName)

  // Step 4: Wait for blockchain to confirm the transaction
  const receipt = await tx.wait()

  // Step 5: Show success with Etherscan link
  addToast('success', 'Item added!', receipt?.hash)
}

Part 3 — Network Safety (Auto-switch to Sepolia)

// Check if MetaMask is on Sepolia testnet
const checkNetwork = async () => {
  const network = await provider.getNetwork()
  const onSepolia = network.chainId === BigInt(11155111)
  setWrongNetwork(!onSepolia) // shows red banner if wrong
}

// One-click switch to Sepolia
const switchToSepolia = async () => {
  await window.ethereum.request({
    method: 'wallet_switchEthereumChain',
    params: [{ chainId: '0xaa36a7' }], // Sepolia in hex
  })
}

// Listen for network changes in real time
useEffect(() => {
  window.ethereum.on('chainChanged', () => checkNetwork())
}, [])

6. App Features

➕

Add Item

Type a product name → click "Add Item to Chain" → MetaMask confirms → item gets a permanent ID on the blockchain. You become the owner.

🔁

Transfer Ownership

Enter item ID + recipient wallet address. Smart contract verifies you are the owner using require(). If yes — ownership transfers permanently.

📦

Browse All Items

App calls itemCount() then loops through all items. Shows name, current owner, and hop count. Completely free — no gas needed.

🔍

Query Item History

Enter any item ID. App calls getHistory() and shows every single wallet address that ever owned the item, in order.
